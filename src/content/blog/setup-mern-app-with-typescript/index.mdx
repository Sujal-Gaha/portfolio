This blog will guide you to setup your own mern app with typescript supported for better development experience.

## 1. Create the Main Project Directory:

```bash
  mkdir [project-name]
  cd [project-name]
```

## 2. Initialize the Root `package.json` File:

```bash
  npm init -y
```

## 3. Create Directories for Backend and Frontend:

```bash
  mkdir backend frontend
```

## 4. Set Up the Backend with Express and TypeScript:

### 4.1. Install the necessary backend dependencies:

```bash
  cd backend
  npm init -y
  npm install express mongoose dotenv
  npm install --save-dev typescript @types/node @types/express @types/mongoose ts-node
```

### 4.2. Initialize TypeScript configuration for the backend:

```bash
  npx tsc --init
```

## 5. Set Up the Backend Folder Structure:

```bash
  mkdir src
  mkdir src/controllers src/models src/routes
  touch src/index.ts
```

## 6. Configure TypeScript for the Backend:

### 6.1. Edit `tsconfig.json` to include:

```json
{
  "compilerOptions": {
    "target": "ES6",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "outDir": "./dist"
  },
  "include": ["src"],
  "exclude": ["node_modules"]
}
```

## 7. Create the Express Server File:

### 7.1. Edit src/index.ts:

```ts
import express from 'express'
import mongoose from 'mongoose'
import dotenv from 'dotenv'

dotenv.config()

const app = express()
const PORT = process.env.PORT || 5000

// Middleware
app.use(express.json())

// Routes
app.get('/', (req, res) => {
  res.send('Hello, MERN App with Next.js!')
})

// MongoDB Connection
mongoose
  .connect(process.env.MONGODB_URI as string, {
    useNewUrlParser: true,
    useUnifiedTopology: true,
  })
  .then(() => console.log('Connected to MongoDB'))
  .catch((err) => console.error('MongoDB connection error:', err))

app.listen(PORT, () => {
  console.log(`Server is running on http://localhost:${PORT}`)
})
```

## 8. Create Environment Variables for the Backend:

### 8.1. Add your MongoDB URI and other environment variables:

```bash
  touch .env
```

### 8.2. Add content to your `.env` file:

```env
  MONGODB_URI=your-mongodb-uri
  PORT=5000
```

## 9. Add Backend Start Script to `backend/package.json`:

### 9.1. Edit package.json in the backend directory:

```json
{
  "scripts": {
    "start": "ts-node src/index.ts"
  }
}
```

## 10. Set Up the Frontend with Next.js and TypeScript:

```bash
  cd ../frontend
  npx create-next-app@latest . --typescript
```

## 11. Configure the Root `package.json` File:

### 11.1. Edit the root package.json:

```json
{
  "name": "mern-app",
  "version": "1.0.0",
  "main": "index.js",
  "license": "MIT",
  "scripts": {
    "dev:backend": "cd backend && npm start",
    "dev:frontend": "cd frontend && npm run dev",
    "dev": "concurrently \"npm run dev:backend\" \"npm run dev:frontend\""
  },
  "devDependencies": {
    "concurrently": "^8.2.0"
  }
}
```

## 12. Run the Application:

### 12.1. Run Backend:

```bash
  npm run dev:backend
```

### 12.2. Run Frontend:

```bash
  npm run dev:frontend
```

## 13. Your Project Structure Should Look Like:

```
    mern-app/
    ├── backend/
    │   ├── src/
    │   │   ├── controllers/
    │   │   ├── models/
    │   │   ├── routes/
    │   │   └── index.ts
    │   ├── .env
    │   ├── package.json
    │   └── tsconfig.json
    ├── frontend/
    │   ├── pages/
    │   ├── public/
    │   ├── styles/
    │   ├── tsconfig.json
    │   └── package.json
    ├── node_modules/
    ├── package.json
    └── README.md
```
